# Elaine Core v0.1 Reviewer Guide

Status: PUBLIC REVIEW MATERIAL / CONTROLLED RETURN PATH AVAILABLE / NOT PRODUCTION READY.

## What You Are Reviewing

Elaine Core v0.1 is a small proof package for a proposed local
cyber-governance framework. It is intended to show how claims, evidence,
limitations, and receipts can be made inspectable before stronger claims are
made.

## What You Are Not Reviewing

You are not reviewing the full private Elaine OS workspace. You are not being
asked to validate production security, compliance, SOC/MDR capability,
deployment readiness, legal posture, patent status, or real-world protection.

## Reviewer Roles

| Role | Main question |
| --- | --- |
| Nontechnical reader | Can you explain the purpose without jargon? |
| Cybersecurity practitioner | Which claims are weak or risky? |
| Developer | Can you run the proof lab without undocumented steps? |
| Skeptical reviewer | What sounds inflated, vague, or confusing? |
| AI governance reviewer | Are human oversight and limits clear? |
| Engineer | Can the package be run and inspected without undocumented help? |
| Creative director | Is the story, tone, trust posture, and onboarding path clear? |
| Production manager | Is the review process executable, trackable, and closable? |

For the first three-reviewer wave, see `FIRST_REVIEW_TEAM_PACKET.md`.

## Review Rules

- Use only the shared clean package and any explicitly attached controlled-alpha receipts.
- Do not request private repo access.
- Do not ask for private lab, employer, business-plan, or patent details.
- Do not reshare the package unless the owner separately authorizes sharing.
- Treat all claims as unproven unless the claim ledger and receipt index support them.

## Minimum Review Output

Provide:

1. a one-sentence explanation of Elaine;
2. the top three confusing points;
3. the top three unsupported or risky claims;
4. the proof-lab run result;
5. whether the limitations were clear;
6. a go/no-go recommendation for wider controlled reviewer testing;
7. a completed `REVIEWER_RETURN_TEMPLATE.json` if you are part of controlled review.

Start with `DOWNLOAD_AND_REVIEW_GUIDE.md` if the package feels too large.
