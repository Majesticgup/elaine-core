# Architecture

Elaine AOS is framed here as a conceptual public-safe architecture for a local cyber-governance orchestrator between a person, their computer, their AI tools, and consequential action.

```text
local request
-> source/evidence discovery
-> claim and data-boundary check
-> gate/authority check
-> local action or hold
-> receipt
-> recall
```

## Public-Safe Components

- Local posture object: a public-safe representation of a local hygiene or governance check.
- Claim object: statement being evaluated.
- Evidence object: support or missing-proof record.
- Gate object: boundary around higher-risk action.
- Receipt object: decision and limitation record.
- Recall pointer: later recovery of proof context.
- Public surface: external-facing wording or artifact.
- LLM boundary: the allowed data and action scope for any model-assisted step.

This architecture document is conceptual. It does not disclose private runtime internals, broker mechanics, internal proof repository schemas, provider routing, or patent-sensitive implementation detail.
