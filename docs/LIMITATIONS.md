# Limitations

- The package uses synthetic examples only.
- It is not a full Elaine OS source release.
- It does not include internal proof repositories, unredacted operational records, private routes, prompts, or runtime internals.
- Apache-2.0 is selected for the public package, but public release is still gated.
- It has not been formally reviewed.
- It has not been counsel reviewed for public IP disclosure.
- It has not been externally validated.
- It has no launch-maturity claim.
- It is not a certification, managed-security, or legal product.
- It has no third-party endorsement claim.
- It does not prove that Elaine secures a real user or a real system.
- It does not replace endpoint security, SIEM, SOC, MDR, identity security, backups, patching, or professional incident response.
- It does not yet include a complete installable local orchestrator.
- Docker proof-runtime files are included as a local proof path, but Docker
  execution receipts are separate evidence unless explicitly bundled.
- Ubuntu baseline evidence does not prove Windows, low-resource, or
  cross-platform runability.
- Windows support remains blocked until a Windows receipt pair passes host
  intake.
- Low-resource compatibility remains blocked until constrained-device receipts
  pass.
- Runtime Core is limited to CLI/status and bounded loopback preview evidence;
  it is not a persistent API/MCP service, model gateway, private RAG system, or
  workstation controller.
- Runtime Core `llm-status` is a held-state report. This package does not
  include or prove a daily-use local LLM assistant, DGX-backed reasoning
  product, Codex replacement, private-corpus reasoning path, hosted provider
  fallback, or autonomous model route.
