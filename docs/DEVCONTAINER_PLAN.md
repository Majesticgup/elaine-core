# Dev Container Plan

Status: CONTRIBUTOR CONVENIENCE PLAN / NOT EXECUTED / NOT REQUIRED FOR REVIEW.

The Docker proof runtime is the default reviewer path. A future Dev Container
can help contributors work on the package, but it should not become a stronger
public claim.

## Planned Role

- open the clean package in a predictable Python environment;
- run the same proof-lab and Runtime Core CLI commands as `QUICKSTART.md`;
- keep proof receipts visible to the contributor;
- avoid private Elaine OS workspace dependencies.

## Current Boundary

No `.devcontainer` runtime is active in this package yet. A future Dev Container
must preserve the same claim ceiling as the Docker proof runtime:

- no provider/API calls;
- no hosted retrieval;
- no private corpus;
- no private workspace mounts;
- no workstation control;
- no compliance, production, or security-benefit claim;
- no public sharing or GitHub action without A5 approval.

## Future Gate

The Dev Container should be added only after the Docker proof runtime has a
passing A3 receipt or the owner explicitly approves a contributor-lane preview.
