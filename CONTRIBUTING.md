# Contributing

Status: DRAFT / CONTRIBUTIONS NOT OPEN

This package is not yet public and is not accepting contributions.

Future contributions should follow proof-before-action and local cyber-governance discipline:

- every material claim needs evidence, limitation, and claim state;
- stronger public wording requires stronger support;
- blocked claims must stay blocked until proof changes;
- synthetic examples must be labeled as synthetic;
- security automation must describe the action, risk, data boundary, rollback, and expected receipt;
- LLM integration examples must show allowed data, blocked data, and human approval points;
- private business, patent-sensitive, employer/customer, secret, account, raw-log, private-route, and private-proof-store material must not be submitted;
- security issues should follow `SECURITY.md`.

No contribution process is active until repository publication and license decisions are approved.
